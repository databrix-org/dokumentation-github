Sicherheitsgruppen einrichten

Sicherheitsgruppen in der Cloud fungieren als virtuelle Firewalls, die den eingehenden und ausgehenden Netzwerkverkehr zu Ihren Instanzen steuern. In diesem Abschnitt richten wir die Sicherheitsgruppen ein, die für die sichere Ausführung unserer E-Learning-Anwendung erforderlich sind.

Neue Sicherheitsgruppe erstellen

Folgen Sie diesen Schritten, um eine neue Sicherheitsgruppe zu erstellen:

  1. Navigieren Sie im linken Menü des bwCloud-Dashboards zu "Network" und dann zu "Security Groups".
  2. Klicken Sie auf die Schaltfläche "Create Security Group".
  3. Geben Sie der Sicherheitsgruppe einen Namen (z.B. "e-learning-security") und eine Beschreibung.
  4. Klicken Sie auf "Create Security Group".

Regeln für die Sicherheitsgruppe hinzufügen

Nach dem Erstellen der Sicherheitsgruppe müssen Sie Regeln hinzufügen, um bestimmte Arten von Netzwerkverkehr zuzulassen:

Eingehende Regeln (Ingress Rules)

Klicken Sie auf der Seite der neu erstellten Sicherheitsgruppe auf "Add Rule" und fügen Sie die folgenden Regeln für eingehenden Verkehr hinzu:

Regel Typ Port CIDR Beschreibung
SSH SSH 22 0.0.0.0/0 Ermöglicht SSH-Zugriff von überall
HTTP HTTP 80 0.0.0.0/0 Ermöglicht HTTP-Zugriff von überall
HTTPS HTTPS 443 0.0.0.0/0 Ermöglicht HTTPS-Zugriff von überall
JupyterHub Custom TCP Rule 8000-8100 0.0.0.0/0 Ermöglicht Zugriff auf JupyterHub und einzelne Notebooks
Docker Swarm Custom TCP Rule 2377 10.0.0.0/8 Docker Swarm Cluster Kommunikation
Docker Registry Custom TCP Rule 5000 10.0.0.0/8 Docker Registry Kommunikation

Warnung: Aus Sicherheitsgründen sollten Sie in einer Produktionsumgebung den Zugriff auf die IP-Adressen einschränken, die Zugriff benötigen, anstatt 0.0.0.0/0 zu verwenden, das Zugriff von überall aus ermöglicht.

Sicherheitsgruppe den Instanzen zuweisen

Nachdem Sie die Sicherheitsgruppe mit den erforderlichen Regeln erstellt haben, müssen Sie sie Ihren VM-Instanzen zuweisen:

  1. Navigieren Sie zu "Compute" und dann zu "Instances".
  2. Klicken Sie für jede Instanz auf das Dropdown-Menü und wählen Sie "Edit Security Groups".
  3. Wählen Sie im Dialogfeld Ihre neu erstellte Sicherheitsgruppe aus und klicken Sie auf "Save".

Hinweis: Stellen Sie sicher, dass alle Ihre VMs (sowohl Manager als auch Worker Nodes) dieselbe Sicherheitsgruppe verwenden, um eine reibungslose Kommunikation innerhalb des Clusters zu gewährleisten.
